PDM is invoked with the `pdm` command. Global flags apply to every subcommand and must be placed before the subcommand name.

## Global flags

| Flag | Description |
|---|---|
| `-V`, `--version` | Show the version and exit |
| `-c`, `--config <file>` | Specify another config file path (env: `PDM_CONFIG_FILE`) |
| `-v`, `--verbose` | Use `-v` for detailed output and `-vv` for more |
| `-q`, `--quiet` | Suppress output |
| `--no-cache` | Disable the cache for the current command |
| `--ignore-python` | Ignore the Python path saved in `.pdm-python` |
| `--pep582` | Enable PEP 582 mode globally |
| `-n`, `--non-interactive` | Don't ask for confirmation |

## pdm init

Initialize a `pyproject.toml` for PDM in the current directory.

```bash
pdm init [OPTIONS] [TEMPLATE]
```

<AccordionGroup>
  <Accordion title="Flags">
    | Flag | Description |
    |---|---|
    | `--copier` | Use Copier to generate the project |
    | `--cookiecutter` | Use Cookiecutter to generate the project |
    | `-n`, `--non-interactive` | Don't ask questions; use default values |
    | `--python <python>` | Specify the Python version or path to use |
    | `--dist`, `--lib` | Create a package for distribution |
    | `--backend <backend>` | Specify the build backend (implies `--dist`). Choices: `pdm-backend`, `setuptools`, `flit`, `hatchling`, `maturin` |
    | `--license <spdx>` | Specify the license (SPDX name) |
    | `--name <name>` | Specify the project name |
    | `--project-version <version>` | Specify the project's initial version |
    | `--no-git` | Do not initialize a git repository |
    | `-r`, `--overwrite` | Overwrite existing files |
    | `TEMPLATE` | Project template — a local path or a Git URL |
    | `--skip <hook>` | Skip running named hook(s) |
  </Accordion>
</AccordionGroup>

<CodeGroup>
```bash Interactive
pdm init
```

```bash Non-interactive with defaults
pdm init --non-interactive --python 3.11 --dist --backend pdm-backend --name mylib
```

```bash From a Git template via Copier
pdm init --copier https://github.com/example/template.git
```
</CodeGroup>

## pdm add

Add package(s) to `pyproject.toml` and install them.

```bash
pdm add [OPTIONS] PACKAGES...
```

<AccordionGroup>
  <Accordion title="Flags">
    | Flag | Default | Description |
    |---|---|---|
    | `-d`, `--dev` | `false` | Add packages to dev dependencies |
    | `-G`, `--group <group>` | — | Target dependency group |
    | `--no-sync` | — | Only write `pyproject.toml`; skip working set sync |
    | `-e`, `--editable <pkg>` | — | Add an editable dependency (`-e path/to/pkg`) |
    | `--save-compatible` | — | Save compatible version specifier (`~=x.y`) |
    | `--save-wildcard` | — | Save wildcard version specifier (`*`) |
    | `--save-exact` | — | Save exact version specifier (`==x.y.z`) |
    | `--save-minimum` | — | Save minimum version specifier (`>=x.y.z`) (default) |
    | `--update-reuse` | — | Reuse pinned versions already present in the lock file |
    | `--update-eager` | — | Update dependencies and their sub-dependencies eagerly |
    | `--update-all` | — | Update all dependencies |
    | `--pre`, `--prerelease` | — | Allow pre-release versions |
    | `-u`, `--unconstrained` | — | Ignore existing constraints from the lock file |
    | `--no-editable` | — | Install non-editable versions |
    | `--no-self` | — | Don't install the project itself |
    | `--dry-run` | — | Show what would be done without modifying anything |
    | `--fail-fast` | — | Stop on first installation failure |
    | `--lockfile <path>` | — | Specify a custom lock file path |
    | `--frozen-lockfile` | — | Don't update the lock file |
    | `--override <req>` | — | Override a dependency version |
    | `--skip <hook>` | — | Skip named hook(s) |
    | `--venv <name>` | — | Run inside a named virtualenv |
  </Accordion>
</AccordionGroup>

<CodeGroup>
```bash Add to default dependencies
pdm add requests
```

```bash Add multiple packages with version constraints
pdm add "flask>=2.0" "sqlalchemy>=2.0,<3.0"
```

```bash Add to dev dependencies
pdm add -d pytest black mypy
```

```bash Add to a named group
pdm add -G docs sphinx sphinx-rtd-theme
```

```bash Add an editable local package
pdm add -e ./path/to/local-pkg
```
</CodeGroup>

## pdm update

  <Accordion title="Flags">

    | `-t`, `--top` | `false` | Only update packages listed in `pyproject.toml` (not transitive) |
    | `--dry-run`, `--outdated` | `false` | Show the diff without modifying the lock file |

    | `--update-all` | default | Update all packages |
    | `--pre`, `--prerelease` | — | Allow pre-release versions |
    | `-u`, `--unconstrained` | — | Ignore version constraints in `pyproject.toml` |
    | `--save-compatible` / `--save-exact` / `--save-minimum` / `--save-wildcard` | — | Version save strategy |
    | `--no-editable` | — | Don't install editable packages |
    | `--no-self` | — | Don't install the project itself |
    | `--fail-fast` | — | Stop on first failure |
    | `--lockfile <path>` | — | Custom lock file path |
    | `--frozen-lockfile` | — | Don't write to the lock file |
    | `--override <req>` | — | Override a dependency |
    | `--skip <hook>` | — | Skip named hook(s) |
    | `--venv <name>` | — | Run inside a named virtualenv |
  </Accordion>
</AccordionGroup>

<CodeGroup>
```bash Update all packages
pdm update
```

```bash Update specific packages

```

```bash Show outdated packages
pdm update --dry-run
```

```bash Update only top-level deps (not transitive)
pdm update --top
```
</CodeGroup>

## pdm sync

Synchronize the current working set with the lock file.

```bash
pdm sync [OPTIONS]
```

<Note>
`pdm sync` always requires a valid, existing lock file. Use `pdm install` if you want PDM to lock and then sync in one step.
</Note>

<AccordionGroup>
  <Accordion title="Flags">
    | Flag | Description |
    |---|---|
    | `-r`, `--reinstall` | Force reinstall of existing dependencies |
    | `-G`, `--group <group>` | Include a dependency group (repeatable) |
    | `-d`, `--dev` / `--no-dev` | Include/exclude dev dependencies |
    | `--default` / `--no-default` | Include/exclude default group |
    | `--clean` | Remove packages not in the lock file |
    | `--only-keep` | Only keep the packages in the selected groups |
    | `--no-editable` | Don't install in editable mode |
    | `--no-self` | Don't install the project itself |
    | `--dry-run` | Show changes without applying |
    | `--lockfile <path>` | Custom lock file path |
    | `--skip <hook>` | Skip named hook(s) |
    | `--venv <name>` | Run inside a named virtualenv |
  </Accordion>
</AccordionGroup>

```bash
pdm sync
pdm sync --clean
pdm sync -G test --no-self
```

## pdm lock

Resolve and lock dependencies into the lock file.

```bash
pdm lock [OPTIONS]
```

<AccordionGroup>
  <Accordion title="Flags">
    | Flag | Description |
    |---|---|
    | `--refresh` | Refresh hashes in the lock file without re-resolving |
    | `--check` | Exit with error if the lock file is out of date |
    | `--update-reuse` | Reuse pinned versions already in the lock file |
    | `--update-reuse-installed` | Reuse installed packages where possible |
    | `--exclude-newer <date>` | Exclude packages newer than this date (`YYYY-MM-DD` or relative like `7d`) |
    | `--python <range>` | Lock for a specific Python range (e.g. `>=3.9`) |
    | `--platform <platform>` | Lock for a specific platform (e.g. `linux`, `windows`, `macos`) |
    | `--implementation <impl>` | Lock for a Python implementation (e.g. `cpython`, `pypy`) |
    | `--append` | Append the result to the current lock file instead of replacing it |
    | `-G`, `--group <group>` | Include dependency group(s) |
    | `--no-default` | Exclude the default group |
    | `--lockfile <path>` | Custom lock file path |
    | `--no-isolation` | Don't isolate the build environment |
    | `--config-setting <key=val>` | Pass config settings to the build backend |
    | `--override <req>` | Override a dependency version |
    | `--skip <hook>` | Skip named hook(s) |
    | `--strategy-change <strategy>` | Change the lock strategy for this run |
  </Accordion>
</AccordionGroup>

<CodeGroup>
```bash Lock all dependencies
pdm lock
```

```bash Verify lock file is current
pdm lock --check
```

```bash Refresh file hashes only
pdm lock --refresh
```

```bash Lock for a cross-platform target
pdm lock --python ">=3.9" --platform linux --append
```
</CodeGroup>

## pdm run

Run commands or scripts with the project's virtual environment loaded.

```bash
pdm run [OPTIONS] [SCRIPT] [ARGS...]
```

<AccordionGroup>
  <Accordion title="Flags">
    | Flag | Description |
    |---|---|
    | `-l`, `--list` | Show all available scripts defined in `pyproject.toml` |
    | `-j`, `--json` | Output all script info as JSON |
    | `-s`, `--site-packages` | Load system site-packages from the selected interpreter |
    | `--recreate` | Recreate the script's isolated environment (for self-contained scripts) |
    | `--skip <hook>` | Skip named hook(s) |
    | `--venv <name>` | Run inside a named virtualenv |
  </Accordion>
</AccordionGroup>

<CodeGroup>
```bash Run a script defined in pyproject.toml
pdm run lint
```

```bash Run an arbitrary command in the project environment
pdm run python -c "import sys; print(sys.version)"
```

```bash List available scripts
pdm run --list
```

```bash Pass extra arguments to a script
pdm run test -k "test_feature"
```
</CodeGroup>

<Tip>
You can also invoke a script directly by name as a shortcut: `pdm lint` is equivalent to `pdm run lint` if `lint` is defined as a script.
</Tip>

## pdm publish

Build and publish the project to PyPI (or another repository).

```bash
pdm publish [OPTIONS]
```

<AccordionGroup>
  <Accordion title="Flags">
    | Flag | Description |
    |---|---|

    | `-P`, `--password <pass>` | Password (env: `PDM_PUBLISH_PASSWORD`) |

    | `-c`, `--comment <text>` | Comment to include with the distribution |
    | `--no-build` | Skip building; upload from `--dest` directly |
    | `-d`, `--dest <dir>` | Directory of artifacts to upload (default: `dist`) |
    | `--skip-existing` | Skip files that already exist in the repository |
    | `--no-very-ssl` | Disable SSL verification |
    | `--ca-certs <path>` | Path to a CA certificate bundle (env: `PDM_PUBLISH_CA_CERTS`) |
    | `--skip <hook>` | Skip named hook(s) |
  </Accordion>
</AccordionGroup>

<CodeGroup>
```bash Publish to PyPI
pdm publish
```

```bash Publish to TestPyPI
pdm publish -r testpypi
```

```bash Publish without rebuilding
pdm publish --no-build -d dist/
```
</CodeGroup>

## pdm venv

Manage virtualenvs for the project. Run `pdm venv <subcommand>`.

```bash
pdm venv [--path <name>] [--python <name>]
pdm venv <subcommand> [OPTIONS]
```

| Top-level flag | Description |
|---|---|
| `--path <name>` | Print the path to the named virtualenv |
| `--python <name>` | Print the Python interpreter path for the named virtualenv |

### pdm venv create

Create a virtualenv.

```bash
pdm venv create [OPTIONS] [PYTHON] [VENV_ARGS...]
```

| Flag | Description |
|---|---|
| `-w`, `--with <backend>` | Backend: `virtualenv`, `venv`, `conda`, or `uv` |
| `-f`, `--force` | Recreate if it already exists |
| `-n`, `--name <name>` | Name the virtualenv |
| `--with-pip` | Install pip into the virtualenv |
| `PYTHON` | Python version or path for the virtualenv |

```bash
pdm venv create
pdm venv create 3.11
pdm venv create --with virtualenv -n dev 3.10
```

### pdm venv list

List all virtualenvs associated with this project.

```bash
pdm venv list
```

### pdm venv remove

Remove a named virtualenv.

```bash
pdm venv remove [OPTIONS] <env>
```

| Flag | Description |
|---|---|
| `-y`, `--yes` | Skip confirmation prompt |

```bash
pdm venv remove dev
pdm venv remove -y 3.11
```

### pdm venv activate

Print the shell command to activate a virtualenv.

```bash
pdm venv activate [env]
```

```bash
# Evaluate the output in your shell:
eval $(pdm venv activate dev)
```

### pdm venv purge

Remove all (or selected) centralised virtualenvs.

```bash
pdm venv purge [OPTIONS]
```

| Flag | Description |
|---|---|
| `-f`, `--force` | Purge all without confirmation |
| `-i`, `--interactive` | Interactively select which to purge |

    |---|---|
    | `PYTHON` | Python version string or path |
    | `-f`, `--first` | Select the first matched interpreter without prompting |
    | `--auto-install-min` | Auto-install the minimum matching version if none is found |
    | `--auto-install-max` | Auto-install the maximum matching version if none is found |
    | `-i`, `--ignore-remembered` | Ignore the previously remembered selection |
    | `--no-version-file` | Don't write a `.python-version` file |
    | `--venv <name>` | Use the interpreter inside the named virtualenv |
    | `--skip <hook>` | Skip named hook(s) |
  </Accordion>
</AccordionGroup>

```bash
pdm use 3.11
pdm use /usr/bin/python3.12
pdm use --first 3.10
```

## pdm export

Export the resolved packages to various formats (e.g. `requirements.txt`).

```bash
pdm export [OPTIONS]
```

| Flag | Description |
|---|---|
| `-f`, `--format <fmt>` | Output format (e.g. `requirements`, `pylock`) |
| `-o`, `--output <file>` | Write output to a file instead of stdout |
| `-G`, `--group <group>` | Dependency groups to include (repeatable) |
| `--no-default` | Exclude the default group |
| `-d`, `--dev` | Include dev dependencies |
| `--no-markers` | Strip environment markers |
| `--no-hashes` | Strip file hashes |
| `--lockfile <path>` | Custom lock file path |

```bash
pdm export -f requirements -o requirements.txt
pdm export -f requirements --no-hashes
```

## pdm import

Import dependencies from another format into `pyproject.toml`.

```bash
pdm import [OPTIONS] <filename>
```

| Flag | Description |
|---|---|
| `-f`, `--format <fmt>` | Source format (auto-detected if omitted) |
| `-d`, `--dev` | Import as dev dependencies |
| `-G`, `--group <group>` | Target dependency group |

```bash
pdm import requirements.txt
pdm import -f pipfile Pipfile
```

## pdm outdated

Check for outdated packages and show the latest available versions.

```bash
pdm outdated [OPTIONS] [PATTERNS...]
```

| Flag | Description |
|---|---|
| `PATTERNS` | Glob patterns to filter which packages to check (e.g. `flask-*`) |
| `--json` | Output results in JSON format |
| `--include-sub` | Include sub-dependencies (transitive), not just direct dependencies |

```bash
pdm outdated
pdm outdated requests flask-*
pdm outdated --json
pdm outdated --include-sub
```

## pdm fix

Detect and fix project problems for the current PDM version.

```bash
pdm fix [OPTIONS] [PROBLEM]
```

| Flag | Description |
|---|---|
| `PROBLEM` | Fix a specific named problem only; omit to fix all |
| `--dry-run` | Show problems without making any changes |

```bash
pdm fix --dry-run   # list problems
pdm fix             # fix all problems
```
