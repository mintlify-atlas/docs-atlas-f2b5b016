This guide walks you through installing PDM, creating a new project, adding dependencies, and running your first script.

## Install PDM

<Steps>
  <Step title="Install PDM">
    Choose your platform:

    <Tabs>
      <Tab title="Linux/macOS">
        ```bash
        curl -sSL https://pdm-project.org/install.sh | bash
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        powershell -ExecutionPolicy ByPass -c "irm https://pdm-project.org/install.ps1 | iex"
        ```
      </Tab>
      <Tab title="pipx">
        ```bash
        pipx install pdm
        ```
      </Tab>
      <Tab title="uv">
        ```bash
        uv tool install pdm
        ```
      </Tab>
      <Tab title="Homebrew">
        ```bash
        brew install pdm
        ```
      </Tab>
      <Tab title="pip">
        ```bash
        pip install --user pdm
        ```
      </Tab>
    </Tabs>

    Verify the installation:

    ```bash
    pdm --version
    ```
  </Step>

  <Step title="Create a new project">
    Use `pdm new` to scaffold a project with a `pyproject.toml`:

    ```bash
    pdm new my-project
    cd my-project
    ```

    PDM will prompt you for the project name, Python version, and other metadata. For an existing project, use `pdm init` instead:

    ```bash
    cd existing-project
    pdm init
    ```
  </Step>

  <Step title="Add dependencies">
    Add packages to your project with `pdm add`:

    ```bash
    pdm add requests
    pdm add "flask>=2.0"
    pdm add pytest -dG test
    ```

    This updates `pyproject.toml` and generates (or updates) `pdm.lock`.
  </Step>

  <Step title="Install dependencies">
    Install all dependencies into your virtual environment:

    ```bash
    pdm install
    ```

    PDM automatically creates a virtualenv in `.venv` if one doesn't exist.
  </Step>

  <Step title="Run your code">
    Run scripts in the project environment:

    ```bash
    pdm run python main.py
    pdm run pytest
    ```

    Or define reusable scripts in `pyproject.toml`:

    ```toml pyproject.toml
    [tool.pdm.scripts]
    start = "python main.py"
    test = "pytest tests/"
    ```

    Then run them with:

    ```bash
    pdm run start
    pdm run test
    ```
  </Step>
</Steps>

## Your first pyproject.toml

Here's what a typical PDM-managed `pyproject.toml` looks like:

```toml pyproject.toml
[build-system]
requires = ["pdm-backend"]
build-backend = "pdm.backend"

[project]
name = "my-project"
version = "0.1.0"
description = "My awesome project"
requires-python = ">=3.9"
dependencies = [
    "requests>=2.28.0",
    "flask>=2.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
]

[tool.pdm.scripts]
start = "python main.py"
test = "pytest"
```

## Common commands

| Command | Description |
|---|---|
| `pdm add <package>` | Add a dependency |
| `pdm remove <package>` | Remove a dependency |
| `pdm update` | Update all dependencies |
| `pdm install` | Install from lock file |
| `pdm run <script>` | Run a script or command |
| `pdm lock` | Regenerate the lock file |
| `pdm publish` | Build and publish to PyPI |
| `pdm python install <version>` | Install a Python interpreter |

<Tip>
Enable shell completion for a better experience. Run `pdm completion bash > ~/.bash_completion` (or the equivalent for your shell).
</Tip>
